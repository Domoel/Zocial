# Zocial 1.1.0

*Released 2026-06-06*

This release centers on a new profile activity bar (posting stats, à la Phanpy), the account join date on profiles, a fixed notification header, and noticeably more robust timeline and logging internals.

## ✨ New features

**Profile: posting activity at a glance**

- A new bar on every profile page (between the stats row and the tabs) shows the ratio of original posts / replies / boosts from the account's last 20 posts — plus "Last N posts in the past X days".
- Hovering the bar always shows the full breakdown (count + percentage for every category); the legend below shows per-category detail.
- Results are cached per session (10-minute freshness, like Phanpy) → no flicker when switching between the profile tabs; a page reload always fetches fresh numbers.
- Ported from Phanpy.

**Profile: join date**

- Shows "Joined <month year>" in the stats row (full date on hover).
- Mobile-friendly layout: all stats incl. the join date fit on one line, wrapping cleanly on very narrow screens.

**Logs**

- Confirmation dialog before "Clear logs", so nothing gets wiped by accident.
- `console.log` is kept in dev builds (stripped only on the production/main channel) → the Logs page shows full debug output in development builds.

## 🐛 Fixes & improvements

**Notifications: correct header for favourites/boosts of replies**

- A favourite, boost or reaction on a post that is itself a reply wrongly showed "reply to …" instead of the actual action. It now correctly shows e.g. "favourited you" / "boosted you". Mentions are unchanged.

**Timeline: fixed occasional blank / stuck timeline**

- Collapsed posts inside bundled threads (rendered at height 0) blocked the virtual list from initializing — the timeline stayed blank despite loaded data and sometimes only "recovered" minutes later. The height invariant now distinguishes "not measured yet" from "measured as 0".

**Logging: more robust and informative**

- The original stack trace is now captured for uncaught errors/promise rejections (visible in-app and in "Copy logs") → the real source instead of a useless handler stack.
- Errors/warnings are prioritized in the log buffer: a flood of `console.log` (e.g. while the timeline loads) no longer pushes them out of the 100-entry buffer.

## 🔧 Internal / build

- The build channel controls `console.log` stripping: main → Production (stripped), dev → Development (kept).
- New non-persisted store cache (per-account posting stats); new API helper for account statuses.
