# Zocial 1.5.0

*Released 2026-06-09*

This update introduces highly anticipated features like quoting posts, crucial accessibility improvements, and noticeable performance upgrades for lists.

## ✨ New Features & Highlights

**Quote Posts**

Zocial now supports quoting posts — completely independent of special backend requirements.

Instead of relying on protocols like FEP-e232 (which is currently only supported by a handful of servers across the Fediverse), Zocial takes a pragmatic, cross-platform approach: The URL of the quoted post is simply appended to your new post. The visual representation is then seamlessly handled by link preview cards. This ensures your quotes look fantastic and native in almost every client across the Fediverse.

**How it works:**

- Access: The quote post feature is always available via the post's context menu, regardless of your backend server capabilities.
- Quick Access (Optional): You can enable a dedicated settings option to use the boost reactions button to either boost or quote post (Settings → Composer).
- Keyboard Shortcut: Simply press `q` on any focused post to quote it.
- Workflow: The composer opens automatically and places your cursor right at the very beginning. This allows you to write your take immediately without needing to manually reposition the cursor first.

## ♿ Accessibility

The user experience has been significantly improved for individuals relying on assistive technologies, as well as for power users who prefer keyboard navigation:

- **Screen readers now read link preview cards:** Previously, screen readers remained silent when navigating past link preview cards in the timeline (e.g., using j/k). Moving forward, the card's title is always included in the post's aria-label.
- **Extended descriptions (Opt-in):** Users who prefer full context can enable a new setting under Settings → Accessibility ("Announce link preview descriptions"). This will read aloud both the title and the description text of the link preview card for full verbosity.
- **New keyboard shortcut for translations:** Pressing "t" on a focused post will now instantly trigger a translation. Previously, this feature was only accessible via a click in the context menu. The in-app keyboard shortcut reference has been updated accordingly.

## ⚡ Performance

- **Immediate timeline rendering for lists:** Load times when opening lists have been drastically reduced. Posts are now rendered immediately on the screen as soon as the network response arrives. Writing to IndexedDB (for offline caching) has been offloaded to a background process. This ensures the page render is no longer blocked, providing a much smoother scrolling experience.

## 🐛 Bug Fixes & Improvements

- **Fixed Friendica list crashes:** Certain server backends, like Friendica, sometimes return list items without a populated account object (e.g., group posts). Previously, this caused a crash cascade that left the entire list timeline blank. All affected code paths now gracefully handle missing account data.
- **Smart List UI:** If a server backend does not support the lists API and returns an error, the "Lists" section and the "Create list" button (under Settings → Community) are now automatically hidden to prevent user confusion.
- **Intelligent Scroll-to-Top Button:** The logic for the scroll-to-top button has been overhauled. It now appears in all timelines that do not have a clickable header tab at runtime. Example: If you have pinned your bookmarks as a tab, the button is hidden there, as you can easily jump to the top by clicking the pinned header icon. If the timeline is not pinned, the button will appear. Additionally, a conflict with uBlock Origin filter lists that were incorrectly hiding the button for some users has been fixed.
- **"Manage list memberships" dialog:** Fixed a bug where the "Manage list memberships" dialog would silently fail and refuse to open in certain situations.
- **Cleaner Logs:** Network errors that occur while offline are now logged at the warn level instead of error. This significantly reduces "noise" on the system logs page.
