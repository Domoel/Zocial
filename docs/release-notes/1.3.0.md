# Zocial 1.3.0

*Released 2026-06-08*

This release adds in-app profile editing, a local-only posting option, and a round of reliability fixes that quiet the error console on mobile (rate-limit / network handling).

## ✨ New features

**Edit your profile, in-app**

You can now edit your own profile directly from your profile page — no page reload, no jumping to the web UI.

- An "Edit profile" button (own profile only) opens a modal to change your display name, bio, up to 4 metadata fields (links), and to upload a new avatar and header image (with live preview).
- Saves via the standard Mastodon `update_credentials` API and updates the profile live. Fully compatible with Mastodon, glitch-soc, Akkoma and GoToSocial.

**"Post to the local timeline only by default"**

- New Composer setting: when enabled, new posts default to local-only on instances that support it.
- The option is automatically greyed out (with an explanatory tooltip) on instances that don't support local-only — so you always see whether it applies.

## 🛠️ Reliability & error handling

A series of fixes that make the app quieter and more robust, especially on mobile/flaky connections:

- **Revoked/expired tokens now log out cleanly:** A 401 on the credentials check is detected by its status code (instead of a stale message-prefix check), so the app logs you out gracefully instead of looping with uncaught errors.
- **API errors are detected by status code, not message text:** As a result: a missing push-notification scope now correctly triggers the re-authentication prompt, a server-side-removed push subscription is cleaned up locally, and other push/network errors are logged instead of silently swallowed.
- **No more uncaught promise rejections from background refreshes:** Cache-first background updates and all per-instance background refreshes (custom emoji, lists, filters, followed hashtags, push, node info, credentials, instance info) are now guarded — a transient 429 rate-limit or a momentary network drop logs a quiet warning instead of flooding the console with uncaught errors.

## 💅 UI polish

- The profile action button (Edit / Follow + "⋯" options) now lives in the top banner, far-left (mirroring the "Back" button), keeping the name/bio area clean and avoiding truncated display names on narrow screens.
- Follow / Edit buttons now have a consistent, fixed height (equal to the options button) and a shared minimum width, for a uniform button design.
