# Zocial 1.6.0

*Released 2026-06-09*

This release reworks the translation backend from the ground up.

## ✨ Translation now uses LibreTranslate

The previous translation feature routed requests through SimplyTranslate as a proxy for Google Translate. SimplyTranslate recently dropped its CORS headers, breaking translation entirely for browser-based clients. Rather than patching around a dependency that relies on Google, the engine was switched to [LibreTranslate](https://libretranslate.com/) — a fully open-source translation engine with no Google dependency.

Translation requests are now routed through the nginx container itself, so no CORS configuration is required and no third-party domain receives your requests unless you explicitly point to one.

**Configurable translation endpoint for self-hosters**

A new `TRANSLATE_API` environment variable lets admins point Zocial at their own LibreTranslate instance. No rebuild required — change `.env`, restart the container, done. The public libretranslate.com instance is used as the default and works without any configuration, though it is rate-limited for heavy use. See `.env.example` for details.

## 🐛 Bug fixes

- Translation errors (rate limits, network failures) now surface a proper error message in the UI instead of a cryptic crash.
- Improved handling for language codes that differ between LibreTranslate and the previous Google-based backend (Hebrew, Javanese, Chinese).
