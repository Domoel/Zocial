# Zocial 1.4.0

*Released 2026-06-09*

This release adds list management directly in the app — creating lists, and adding/removing accounts from them — along with a scroll-to-top button on profiles, and a round of reliability fixes mostly affecting GoToSocial users.

## ✨ New features

**Add/remove accounts from lists — right from the profile**

You can now manage list memberships without leaving a profile:

- The "···" menu on any profile you follow shows a new Manage list memberships entry (visible only when you follow the account and have at least one list).
- Tapping it opens a dialog listing all your lists, with checkmarks for current membership. Tap any list to toggle.
- On servers that don't support the membership API the dialog shows a clear "Not supported by this server" notice instead of a broken empty state.

**Create lists directly in Zocial**

A Create list button now appears at the bottom of the Lists section in Community Settings. This closes the chicken-and-egg problem for GoToSocial users, who have no separate web UI to create lists through. After creation the list appears immediately and can be pinned as a timeline tab.

**Scroll-to-top button on profile timelines**

Profile timelines (Posts, Posts & Replies, Media) now show the same scroll-to-top button that thread views already had. It appears once you scroll past the "Last 20 posts in the past X days" bar, and works identically across all three tabs regardless of whether the account has pinned posts.

## 🛠️ Reliability & bug fixes

- **GoToSocial: timeline crash on posts with a missing URL** — GoToSocial can return posts where `url` is null. `new URL(null)` throws inside a timeline component computed, cascading into a series of unrelated-looking errors (parentNode, src, emojis). The URL is now resolved with a try/catch — falling back to the ActivityPub `uri`, then to a plain status route — so affected posts load instead of crashing the whole timeline.
- **GoToSocial: false "offline" toast for empty list timelines** — GoToSocial returns an HTTP error (rather than an empty array) when a list has no members. The generic catch block treated this as a network failure and showed "Internet request failed. Showing offline content." HTTP errors on list timelines are now handled silently as empty results.
- **Back button in profile stopped working** — The scroll-to-top button is sticky-positioned at the top-right with `z-index: 5000`, the same position as the Back button in the banner. Its inner button element retained `pointer-events: auto` even when invisible (opacity 0), silently intercepting Back clicks. Fixed: the explicit `pointer-events: auto` is removed so the button inherits `none` from its parent when hidden.
- **"Manage list memberships" dialog sometimes did not open** — A race condition between dialog close and open: `history.back()` from the closing options dialog could fire a popstate event after the new dialog had registered its listener but before it had called `pushState`. The new dialog saw its own ID missing from history state and closed itself immediately. Fixed by following the established codebase pattern — import the module first, then close the current dialog, then open the new one.
- **Swipe gesture broken on the Mentions tab** — The horizontal swipe gesture for switching between top-level nav tabs did not work from the Mentions tab because the internal page name `notifications/mentions` wasn't being normalised before the nav lookup. Fixed.
- **GoToSocial: follower count decreasing after every profile save** — Saving profile changes overwrote the displayed follower/following/post counts with values from `PATCH /accounts/update_credentials`, which GoToSocial reports lower than `GET /accounts/:id`. Profile counts are now preserved from the current view and only credential fields are updated from the PATCH response.

## ⚙️ Internal

- Background per-instance refreshes (emoji, lists, filters, followed hashtags, push, node info, credentials, instance info) are now guarded against running for instances that have been logged out between trigger and callback — eliminating a class of stale writes and potential console noise.
- Null guard added to `getAccountAccessibleName` to prevent crashes when account data isn't yet available.
- New `PageListActionItem` component: a `<button>`-based counterpart to `PageListItem` for settings entries that trigger an action rather than navigate to a route.
