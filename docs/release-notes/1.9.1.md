# Zocial 1.9.1

## ✨ Features

- **Remove a follower** — the account "⋯" menu now has a "Remove {account} from followers" entry for anyone who follows you (Mastodon's `remove_from_followers`). It shows on every row of the **Followers** list and on the profile of anyone who follows you.
- **Asks for confirmation first.** Unlike block or mute — which you can undo yourself — removing a follower can only be undone by the *removed person* (they'd have to follow you again). A misclick would mean having to reach out to them, so this action confirms before going through, even though Zocial's other menu actions don't.
- **Works on any backend that supports it, automatically.** Not every server implements this endpoint (e.g. GoToSocial currently doesn't). Rather than maintaining a per-backend support list, Zocial just tries it and shows a clear "this server doesn't support removing followers" message if it isn't available — so the feature lights up by itself the moment any backend adds support, with no app update needed.

## 🧹 Internal

- New strings localised in all five languages. Backend support is handled by graceful 404/501 degradation rather than capability allow-listing (no Fediverse capability flag exists for this endpoint, and a manual allowlist isn't maintainable).
