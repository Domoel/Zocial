# Zocial 1.9.1

## ✨ Features

- **Remove a follower** — the account "⋯" menu now has a "Remove {account} from followers" entry for anyone who follows you (Mastodon's `remove_from_followers`). It shows on every row of the **Followers** list and on the profile of anyone who follows you.
- **Asks for confirmation first.** Unlike block or mute — which you can undo yourself — removing a follower can only be undone by the *removed person* (they'd have to follow you again). A misclick would mean having to reach out to them, so this action confirms before going through, even though Zocial's other menu actions don't.
- **Hidden where it isn't supported.** The entry only appears on backends that actually implement the endpoint (Mastodon and Mastodon forks). On GoToSocial and other backends that don't support it, the entry is simply not shown — no dead menu item. Detection uses the server's own software identity (nodeInfo), so it's reliable; a safety-net message still covers any edge case.

## 🧹 Internal

- Backend capability detection for `remove_from_followers` via the authoritative nodeInfo `software.name` (`supportsRemoveFromFollowers` computed), rather than fragile version-sniffing. New strings localised in all five languages.
