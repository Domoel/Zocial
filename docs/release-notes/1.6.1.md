# Zocial 1.6.1

*Released 2026-06-10*

Improvements and bug fixes to the translation feature introduced in 1.6.0.

## ✨ New

- **"Post is already in your language" message** — When you trigger translation on a post that's already in your browser language, Zocial now detects this and shows a friendly message instead of attempting a translation that can only fail. The detection happens in parallel with the translation request, so there's no extra round-trip for the common case.

## 🐛 Bug fixes

- **"Translated from Detect language" shown instead of the actual language** — The detected source language is now read directly from the `/translate` response, where LibreTranslate includes it when `source: auto` is set. Previously a separate `/detect` call was silently failing.
- **Language code mismatches for Hebrew, Javanese, and Chinese** — LibreTranslate uses different codes than the previous backend for `he`, `jv`, and `zh`. Without the mapping, the language name displayed as `undefined`.
- **Container crash when translation backend is unreachable** — nginx previously resolved the upstream hostname at startup. If the backend was down, the container refused to start. Fixed with a runtime variable + Docker's embedded DNS resolver.
- **Translation target defaulted to build-time locale** — A German user on the default image (built with `LOCALE=en-US`) would always get posts translated into English. The target language is now taken from `navigator.language` at runtime.
- **Unhelpful error on non-JSON responses** — Error messages now include the HTTP status code and the first 200 characters of the actual response, making misconfiguration much easier to diagnose.
- **Trailing slash in `TRANSLATE_API` caused routing failures** — Stripped automatically in `entrypoint.sh`.
- **Correctly detect same-language and fix Svelte 2 template syntax** — Check `detectedLanguage` on a successful translate response (not only on failure).

## ⚡ Performance

- Translation detect and translate calls now run in parallel. HTML content is passed with `format: html` so LibreTranslate preserves markup. Only plain text is sent to `/detect` (HTML stripped beforehand).
